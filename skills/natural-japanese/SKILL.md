# natural-japanese

日本語の文章から「AI臭さ」を取り除き、自然な文章に仕上げるスキル。対象は note・ブログ・エッセイ・技術文書など全般。

## 設計思想

AI は自分自身の AI 臭さを認識しにくい。だから修正の前に、まず機械で決定的に検出する。検出後にその指摘をどう直すかは AI（あなた）が判断する。「検出は機械、判断は人間（またはAI）」という役割分担がこのスキルの軸になる。

## 0. まなざし（文体プロファイル）— 任意

`style-profile.md`（プロジェクトルートかユーザー指定の場所）が既にあれば、執筆・リライトの前に読み込み、そこに書かれた視点・語彙・リズムの癖を下敷きにする。

なければ**汎用モードで進めてよい**。プロファイルは必須ではなく、AI臭除去そのものは汎用モードでも機能する。

ユーザーが「自分の文体を学ばせたい」「プロファイルを作りたい」と求めた場合のみ、セットアップフローに入る。

### セットアップフロー（求められたときだけ）

1. ユーザーの過去文章を3〜5本読む（note・ブログ・過去原稿など、ユーザーに場所を尋ねる）
2. `assets/style-profile-template.md` の観察項目（視点・文末の癖・リズム・語彙・構成）に沿って特徴を抽出する
3. テンプレートを埋めた `style-profile.md` を書き出す。断定しすぎず、「傾向として」と留保をつけて書く（3〜5本という少ない標本で言い切ると外れやすい）

## 1. 執筆前の設計 — 揺らぎ

下書きに入る前に、扱うトピックの重要度に意図的なムラをつける。すべての段落を同じ熱量・同じ厚みで書くと、それ自体が「整いすぎた不自然さ」になる。詳しい手順は `references/revision-guide.md` の「揺らぎ設計」を参照。

## 2. 下書き

プロファイル（あれば）と揺らぎ設計を踏まえて書く。この段階では禁止語やリズムを気にしすぎず、まず内容を出し切ってよい。

## 3. 静的検知

```
uv run scripts/ai-smell-lint.py --json <file>
```

このスクリプトは禁止語・翻訳調・否定肯定対比の反復・文長の均質さ・体言止め率・段落頭の接続詞率・語彙多様性・英語統語の疑いなどを機械的に検出する。exit code は常に 0（CI ゲートではなく lint）。

`uv` が使えない環境（Claude.ai 等）では、代わりに `references/manual-checklist.md` の文章版チェックリストで同じ観点を人手でなぞる。

## 4. 検出結果の判断

lint の `findings` はあくまで疑いの提示であり、機械的に全部直せという指示ではない。各カテゴリの意味と対応方針は `references/revision-guide.md` の対応表を見て、文脈に照らして「直す/直さない」を判断する。

用語カタログが必要なら:
- 禁止語・LLM常套句の詳細と理由 → `references/forbidden-patterns.md`
- 英語統語・翻訳調のパターンと書き直し例 → `references/translationese.md`

## 5. 自己点検ループ

修正後、一度「これは他人が読んで指摘するとしたら」という視点に立ち戻って読み直す。手順は `references/revision-guide.md` の「自己点検ループ」を参照。

## 6. 再検知・収束

3〜5 を最大2〜3周繰り返す。周を重ねても同じカテゴリの指摘が消えない場合は、機械検出の閾値の問題か、文脈上そのままでよい表現かを見極めて止める。無限に回さない。
